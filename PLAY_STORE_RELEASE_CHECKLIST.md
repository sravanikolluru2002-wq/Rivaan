# Rivan Realty Google Play Release Checklist

This repo already contains a Capacitor Android app at `frontend/android`. Use that path for the first Play Store release because it packages the same production web app used at `rivanreality.com`.

## Current Production Baseline

- Android package ID: `com.rivan.reality`
- App name: `Rivan Realty`
- Minimum SDK: `24`
- Compile SDK: `36`
- Target SDK: `36`
- Release artifact: Android App Bundle (`.aab`)
- Web source: `frontend/dist`, generated by `npm --prefix frontend run build`
- Privacy policy path: `https://rivanreality.com/privacy.html`

## One-Time Play Console Setup

1. Create the app in Google Play Console with package ID `com.rivan.reality`.
2. Enable Play App Signing.
3. Generate a private upload key and keep it outside Git.
4. Add store listing text, screenshots, feature graphic, app category, contact details, and privacy policy URL.
5. Complete Data Safety based on actual app data collection: account/contact details, property enquiries, visits, bookings, CRM/admin activity, diagnostics, and authentication data.
6. Complete content rating and target audience forms.
7. Add testers for internal testing before production rollout.

## Required Local Signing Variables

Set these only on the build machine or CI secret store:

```powershell
$env:RIVAN_UPLOAD_STORE_FILE="C:\secure\rivan-upload-key.jks"
$env:RIVAN_UPLOAD_STORE_PASSWORD="..."
$env:RIVAN_UPLOAD_KEY_ALIAS="rivan-upload"
$env:RIVAN_UPLOAD_KEY_PASSWORD="..."
```

Do not commit `.jks`, `.p12`, `.key`, `.pem`, or local `.env` files.

## Build Commands

```powershell
npm run build
npm run android:sync
npm run android:bundle
```

The release bundle is generated at:

```text
frontend/android/app/build/outputs/bundle/release/app-release.aab
```

## Pre-Submission Smoke Test

- Fresh install opens without a white screen.
- Customer OTP login restores the session after closing and reopening the app.
- Customer home images load with placeholders and no broken price text.
- Customer can schedule a visit and see it in Site Visits.
- Partner login shows Partner labels, saved partner profile name, assigned visits, bookings, properties, and notifications.
- Admin login shows users, partner approvals, bookings, visits, properties, notifications, audit, settings, and profile without horizontal overflow.
- WebSocket failure does not block the app; API fallback still refreshes dashboards.
- Offline banner appears when network is unavailable and disappears after reconnection.
- Back button behavior does not trap users on login/dashboard screens.
- Privacy policy opens at `/privacy.html`.

## Release Guardrails

- Increment `versionCode` for every Play upload.
- Increment `versionName` for public releases.
- Never ship with debug signing.
- Never add broad Android permissions unless a real feature requires them.
- Keep production demo/seed data disabled.
- Roll out first to internal testing, then closed testing, then staged production.
